# 注释规范

1. **详尽性**：尽可能为每一行内容添加注释
2. **自明性**：注释应独立自明，不依赖上下文
3. **一致性**：所有文件格式的注释风格应保持风格一致

## 注释分类

### 1. 文件头

使用多行注释对文件名、文件内容的简要描述、与文件关联的其他文件的路径进行记录，无多行注释的文件格式连续使用单行注释。

```example_for_jsonc
/* {{文件名}} - {{简要描述}}
 *
 * - {{关联文件路径}}/{{文件名}}
 * - path/to/file2.ts */
```

### 2. 模块与子模块

- 使用多行注释分隔不同模块并根据模块内容设置标题；
- 无多行注释的文件格式应额外加入对称的结尾符；
- 使用 `===` 将模块标题注释行长度统一在 32~48 字符之间；
- 使用 `---` 将子模块标题注释行长度统一在 15~25 字符之间。

```example_for_html
<!-- ========== {{模块标题}} ========== -->
```

```example_for_yaml
# ---- {{子模块标题}} ---- #
```

### 3. 行尾说明

优先使用单行注释在行内末尾对主体内容进行解释说明。不支持行内注释的文件在上一行添加单行注释。

```example_for_ts/js
{{主体内容}} // {{对主体内容的作用、目的、生效模式进行说明}}
```
